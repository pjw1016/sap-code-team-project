## 3. 현재 화면 구조

현재 화면은 최상위가 `sap.f.FlexibleColumnLayout`이다. 기존에는 전체 DynamicPage 안에 FCL이 들어가는 형태였으나, 사용자 피드백에 따라 FCL이 페이지 전체 기준으로 동작하도록 변경했다.

```text
Main.view.xml
└─ sap.f.FlexibleColumnLayout
   ├─ Begin Column
   │  └─ sap.uxap.ObjectPageLayout
   │     ├─ Header: 조회조건
   │     └─ Section: KPI + RFQ Header 목록
   └─ Mid Column
      └─ sap.uxap.ObjectPageLayout
         ├─ Header: 선택 RFQ 요약 정보
         ├─ Section 1: RFQ Item 목록
         ├─ Section 2: 금액 비교 차트
         └─ Section 3: MQ 비교 목록
```

Mid Column은 기존 `IconTabBar` 전환 방식이 아니다. SAPUI5 SDK의 ObjectPageLayout 샘플 스타일을 참고해서 `ObjectPageSection`을 세로로 나열하고, AnchorBar를 누르면 해당 섹션으로 자동 스크롤되는 방식으로 바꿨다.

| EntitySet | 목적 | 현재 UI5 연결 |
|---|---|---|
| `RFQHeaderSet` | RFQ Header 목록과 Header 상태 조회 | 연결 완료 |

중요 호출 예시는 다음과 같다.

```text

```

```text
/sap/opu/odata/SAP/ZGWD3MM0005_SRV/RFQItemSet?$filter=RfqNo eq 'RQ00000001'&$format=json
```

```text
/sap/opu/odata/SAP/ZGWD3MM0005_SRV/MQCompareSet?$filter=RfqNo eq 'RQ00000001' and RfqItem eq '00010'&$format=json
```

```text
/sap/opu/odata/SAP/ZGWD3MM0005_SRV/MQDetailSet(MqNo='MQ00000001',MqItem='00010')
```

```text
MERGE /sap/opu/odata/SAP/ZGWD3MM0005_SRV/QuotationItemSet(MqNo='MQ00000001',MqItem='00010')
Body:
{
  "MqNo": "MQ00000001",
  "MqItem": "00010",
  "ActionType": "AWARD"
}
```

채택취소는 `ActionType`을 `CANCEL`로 보낸다.

## 9. Backend 업무 규칙 요약

DPC_EXT에서 확인한 핵심 업무 규칙이다.

| 구분 | 규칙 |
|---|---|
| Header 상태 | `N` 미채택, `P` 일부채택, `A` 채택, `PO` PO생성 |
| RFQ Item 상태 | `N` 미채택, `A` 채택, `PO` PO생성 |
| MQ 응답 판단 | `NETPR <= 0` 또는 `MENGE <= 0`이면 미응답 |
| MQ 추천 | 유효 MQ 중 KRW 환산 총액이 가장 낮은 건, 동률이면 납기일 빠른 건 |
| MQ 선택 불가 | 미응답 MQ, 이미 PO 생성된 MQ, 이미 현재 채택된 MQ |
| 채택 처리 | 같은 RFQ Item의 기존 채택 MQ를 해제한 뒤 선택 MQ만 `SELIDC = X` |
| 채택취소 처리 | 현재 채택된 MQ만 취소 가능 |
| PO 생성 후 변경 | 같은 RFQ Item에 PO가 있으면 채택/취소 모두 불가 |
| 환율 | FI CBO `ZTD3FI0008`, 기준일은 `SY-DATUM`, KRW 환산은 Backend에서 계산 |

UI5는 이 규칙을 다시 계산하지 말고 Backend 응답 필드를 화면 표시와 버튼 활성화에 사용해야 한다.

## 13. 주의할 점

- `sap.m.Table`과 `sap.ui.table.Table`을 혼용하는 현재 설계는 유지한다.
  - RFQ Header, RFQ Item은 반응형 목록 성격이 강하므로 `sap.m.Table`.
  - MQ 비교 목록은 컬럼 수가 많고 비교표 성격이 강하므로 `sap.ui.table.Table`.
- Mid Column은 `IconTabBar` 방식으로 되돌리지 않는다.
  - 현재 사용자 피드백은 세로 섹션 나열 + AnchorBar 스크롤 방식이다.
- Search Help는 지금 구현하지 않는다.
  - 이전 납기 지연 앱처럼 CDS View를 만든 뒤 연결하는 것이 사용자 결정사항이다.
- OData 오류는 `MessageToast`만으로 끝내지 말고 이후 `MessagePopover`로 모아야 한다.
- Backend가 이미 상태와 선택 가능 여부를 계산하므로 UI5에서 중복 판단을 많이 만들지 않는다.

현재 개발은 RFQ Header 조회와 RFQ Item 조회까지 연결된 상태다. 다음 개발은 RFQ Item 선택 이벤트를 만들고 `/MQCompareSet`을 연결해서 MQ 후보 비교표와 차트 데이터를 채우는 단계부터 시작하면 된다.
