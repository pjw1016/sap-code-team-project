## 2. 현재 앱 구현 상태

현재 `quote_comparison` 앱은 RFQ 조회부터 MQ 비교, MQ 상세조회, 채택, 채택취소까지 1차 핵심 흐름이 구현된 상태다.

| 영역 | 현재 상태 |
|---|---|
| Begin Column | 조회조건, KPI 요약, RFQ Header 목록 |
| Mid Column | 선택 RFQ Header, RFQ Item 목록, 금액 비교 차트, MQ 비교 목록 |
| RFQ Header 조회 | `RFQHeaderSet` 조회 후 KPI와 Header 목록 갱신 |
| RFQ Item 조회 | Header 선택 후 `RFQItemSet` 조회 |
| MQ 비교 | RFQ Item 선택 후 `MQCompareSet` 조회 |
| MQ 상세 | 행별 상세 아이콘 또는 선택 MQ 상세 버튼으로 `MQDetailSet` 조회 |
| 거래선 채택 | `QuotationItemSet` MERGE, `ActionType = AWARD` |
| 채택취소 | `QuotationItemSet` MERGE, `ActionType = CANCEL` |
| 차트 | `NetwrKrw` 기준 금액 비교 차트 |
| 자동추천 | Backend의 `RecommendYn`, `CanSelect` 기준으로 선택 가능 추천 MQ 지정 |
| Search Help | 아직 미구현. CDS View 준비 후 `TableSelectDialog` 방식 권장 |
| PO 생성 | 아직 미구현. Backend 설계/SEGW 확장 후 2차 기능으로 진행 |

## 3. 이번 학원 마무리 개발에서 반영한 주요 변경

### 3.1 Begin Column RFQ Header 목록 컬럼 정리

수정 파일: `webapp/view/Main.view.xml`

RFQ Header 목록의 RFQ 번호 아래에 회사코드, 구매조직, 구매그룹이 작은 글씨로 한 줄에 붙어 있던 구조를 분리했다.

변경 후 구조:

| 컬럼 | 표시 방식 |
|---|---|
| RFQ 번호 | `RfqNo`만 표시 |
| 회사코드 | 큰 글씨 `Bukrs`, 작은 글씨 `Butxt` |
| 구매조직 | 큰 글씨 `Ekorg`, 작은 글씨 `Ekotx` |
| 구매그룹 | 큰 글씨 `Ekgrp`, 작은 글씨 `Eknam` |

사용자가 회사/구매조직/구매그룹을 별도 기준으로 보기 쉬워졌고, 향후 정렬/그룹 기준과도 맞는다.

### 3.2 Mid Column RFQ Item 목록 중복 컬럼 제거

수정 파일: `webapp/view/Main.view.xml`

RFQ Item 목록에서 `자재명` 컬럼을 제거했다.  
이미 `자재코드` 컬럼의 `ObjectIdentifier` 안에서 큰 글씨 `Matnr`, 작은 글씨 `Maktx`로 표시하고 있으므로 중복 표기를 없앤 것이다.

### 3.3 Mid Dynamic Header 명칭 표시

수정 파일: `webapp/view/Main.view.xml`

Mid Column Header에서 회사코드, 구매조직, 구매그룹이 코드만 표시되어 사용자가 의미를 알기 어려웠다.

현재 표시 방식:

```text
회사코드
1000 (삼만리 모빌리티)

구매조직
2000 (구매조직명)

구매그룹
002 (구매그룹명)
```

명칭 필드가 비어 있으면 기존처럼 코드만 표시되도록 expression binding을 사용했다.

### 3.4 MQ 비교 목록 정렬 기준 변경

수정 파일: `webapp/controller/Main.controller.js`

`MQCompareSet` 조회 결과는 화면에 표시하기 전에 `NetwrKrw` 기준 오름차순으로 정렬한다.

관련 메서드:

| 메서드 | 역할 |
|---|---|
| `_sortMqCompareRowsByNetwrKrw` | MQ 비교 행과 차트 행을 KRW 환산총액 오름차순 정렬 |
| `_getNetwrKrwSortValue` | 정렬용 숫자값 반환. 숫자가 아니면 맨 뒤로 보내기 위해 무한대 처리 |
| `_prepareChartRows` | 차트에 표시할 응답 MQ만 추출하고 동일 정렬 적용 |

이 변경으로 MQ 비교 테이블과 차트가 같은 순서를 가진다.  
사용자는 가장 낮은 환산총액부터 위에서 아래로, 차트에서는 왼쪽에서 오른쪽으로 비교할 수 있다.

### 3.5 금액 비교 차트 X축 표시 변경

수정 파일:

| 파일 | 변경 |
|---|---|
| `webapp/controller/Main.controller.js` | 차트 행에 `Lifnr` 추가 |
| `webapp/view/Main.view.xml` | 차트 Dimension value를 `Name1`에서 `Lifnr`로 변경 |

기존에는 차트 X축에 공급업체명이 표시되어 이름이 길면 보기 불편했다.  
현재는 공급업체 코드만 표시한다.

```text
변경 전: Shenzhen Battery Pack Ltd.
변경 후: V00006
```

### 3.6 MQ 비교 목록 공급업체 컬럼 표시 변경

수정 파일:

| 파일 | 변경 |
|---|---|
| `webapp/view/Main.view.xml` | 공급업체 셀 title/text 순서 변경 |
| `webapp/i18n/i18n.properties` | `vendor=공급업체` 추가 |

현재 표시 방식:

| 컬럼명 | 큰 글씨 | 작은 글씨 |
|---|---|---|
| 공급업체 | `Lifnr` | `Name1` |

기존에는 컬럼명이 `공급업체명`이고 큰 글씨가 공급업체명이었으나, 사용자가 코드를 우선 확인하기 좋게 바꿨다.

### 3.7 MQ 비교 목록 컬럼 축소

수정 파일: `webapp/view/Main.view.xml`

컬럼이 너무 많아 가로 스크롤이 길어지는 문제가 있어 다음 컬럼을 화면에서 제거했다.

| 제거 컬럼 | 제거 이유 |
|---|---|
| 응답상태 | Radio 활성/비활성, 상세조회, Backend 상태로 간접 판단 가능 |
| 선택 가능 | Radio 비활성 및 tooltip으로 판단 가능 |

중요: 화면 컬럼만 제거했다.  
`ResponseStatus`, `CanSelect` 데이터와 선택 가능 판단 로직은 Controller에서 계속 사용한다.

### 3.8 적용환율 컬럼 추가

수정 파일:

| 파일 | 변경 |
|---|---|
| `webapp/view/Main.view.xml` | `단가`와 `KRW 환산총액` 사이에 `적용환율` 컬럼 추가 |
| `webapp/i18n/i18n.properties` | `kursf=적용환율`로 변경 |
| `webapp/model/formatter.js` | 환율 전용 formatter 추가 |
| `webapp/fragment/MQDetailDialog.fragment.xml` | 상세 팝업 환율 표시도 같은 formatter 적용 |

`MQCompare` metadata에 이미 `Kursf` 필드가 있어서 SEGW 수정 없이 프론트 컬럼만 추가했다.

환율 표시 규칙:

| 값 | 표시 |
|---|---|
| `1507.9` | `1,507.90` |
| `1507.98765` | `1,507.98765` |

`formatAmount`는 금액용이라 끝자리 0을 생략한다.  
환율은 소수점 자리 자체가 의미 있으므로 `formatExchangeRate`를 별도로 만들었다.

### 3.9 MQ 상세조회 팝업 제목 및 레이아웃 변경

수정 파일: `webapp/fragment/MQDetailDialog.fragment.xml`

제목 변경:

```text
변경 전: MQ 상세조회
변경 후: MQ 상세조회 - MQ00000001
```

레이아웃 변경:

| 변경 전 | 변경 후 |
|---|---|
| `SimpleForm` 기반 1열처럼 표시 | `FlexBox wrap` 기반 2열 배치 |
| 왼쪽에만 값이 몰리고 오른쪽 여백이 큼 | 라벨/값 한 쌍을 `27rem` 단위로 묶어 좌우 배치 |
| 팝업 폭 활용이 약함 | 기본정보, 견적정보, 참조정보, 상태정보가 더 촘촘하게 표시 |

`SimpleForm columnsL/M`만으로는 의도한 2열이 적용되지 않았다.  
원인은 섹션마다 FormContainer가 하나라 실제 화면에서 1열처럼 배치됐기 때문이다.  
따라서 명시적인 `FlexBox wrap` + `HBox` 구조로 변경했다.

## 5. 현재 OData 사용 현황

| EntitySet | 사용 목적 | 주요 사용 위치 |
|---|---|---|
| `RFQHeaderSet` | RFQ Header 목록, KPI 계산 | `onSearch`, `_loadRfqHeaders` |
| `RFQItemSet` | 선택 RFQ의 Item 목록 | `_loadRfqItemsForHeader` |
| `MQCompareSet` | 선택 RFQ Item의 MQ 후보 비교 | `_loadMqCompareForRfqItem` |
| `MQDetailSet` | MQ 상세 팝업 데이터 | `_loadMqDetail` |
| `QuotationItemSet` | 채택/채택취소 MERGE | `_updateQuotationItemAward` |

채택 처리 Payload:

```json
{
  "ActionType": "AWARD",
  "RfqNo": "...",
  "RfqItem": "...",
  "MqNo": "...",
  "MqItem": "..."
}
```

채택취소 처리 Payload:

```json
{
  "ActionType": "CANCEL",
  "RfqNo": "...",
  "RfqItem": "...",
  "MqNo": "...",
  "MqItem": "..."
}
```

## 9. 집에서 이어서 개발할 때 주의할 점

1. 이번 변경은 화면 UX 보정이 많다. 이전 상태로 되돌리면 사용자가 확인한 UI가 깨질 수 있다.
2. `CanSelect`, `ResponseStatus` 컬럼은 화면에서만 제거했다. 로직에서는 계속 필요하므로 모델 필드나 formatter를 삭제하면 안 된다.
3. `Kursf`는 `MQCompare`와 `MQDetail` metadata에 이미 존재한다. 적용환율 컬럼을 위해 SEGW 수정은 현재 필요 없다.
4. 환율은 `formatAmount`가 아니라 `formatExchangeRate`를 써야 한다.
5. MQ 상세 팝업은 `SimpleForm`을 사용하지 않는다. 다시 `SimpleForm`으로 되돌리면 1열 배치 문제가 재발할 가능성이 높다.
6. 차트와 MQ 비교 테이블은 `NetwrKrw` 오름차순을 맞춰야 한다. 둘 중 하나만 정렬하면 사용자 혼란이 생긴다.
7. 주석은 한국어 기준으로 유지한다. 도구 지시 주석만 영어 문법을 허용한다.
