## 3. 이번에 추가 수정한 내용

### 3.1 Begin 컬럼 KPI/테이블 블록 분리

수정 파일: `webapp/view/Main.view.xml`

기존에는 `RFQ KPI 요약`과 `RFQ 헤더 목록`이 같은 흰 영역 안에서 좌우 배치처럼 렌더링되어 테이블 컬럼이 좁아졌다.  
이를 해결하기 위해 KPI 영역과 RFQ Header Table 영역을 별도 `ObjectPageSubSection`으로 분리하고, 각각 `sap.m.Panel`로 감쌌다.

구조는 다음과 같다.

```text
ObjectPageSection
  ObjectPageSubSection
    Panel
      RFQ KPI 요약
  ObjectPageSubSection
    Panel
      RFQ 헤더 목록 sap.m.Table
```

CSS를 새로 만들지 않고 SAPUI5 표준 컨트롤인 `sap.m.Panel`, `sap.m.OverflowToolbar`, `sap.m.GenericTile`, `sap.m.Table`만 사용했다.

### 3.2 KPI 빠른 필터 방식 변경

수정 파일: `webapp/controller/Main.controller.js`

기존 KPI 클릭 로직은 `filter>/AwardStatus`를 바꾸고 `onSearch()`를 호출했다.  
이 방식은 KPI 숫자까지 다시 계산되어 사용자가 조회 버튼으로 본 전체 요약이 사라지는 문제가 있었다.

현재 로직은 다음과 같다.

```text
KPI 카드 클릭
  -> 조회조건 모델은 변경하지 않음
  -> RFQHeaderSet 재조회하지 않음
  -> idRfqHeaderTable의 items binding에만 AwardStatus Application filter 적용
  -> KPI 숫자는 마지막 조회 결과 기준 유지
  -> RFQ 헤더 목록 (N)은 현재 화면에 보이는 행 수 기준으로 갱신
```

관련 메서드:

| 메서드 | 역할 |
|---|---|
| `onKpiAwardStatusPress` | KPI 카드 클릭 이벤트 |
| `onClearAwardStatusQuickFilter` | KPI 빠른 필터 해제 |
| `_applyHeaderQuickAwardStatusFilter` | sap.m.Table binding에 Application filter 적용 |
| `_updateRfqHeaderCountForQuickFilter` | 테이블 제목의 표시 건수 갱신 |
| `_getHeaderTableStatusSummary` | 현재 상태 표시 문구 계산 |

## 5. Backend OData 기준

현재 UI5는 아래 EntitySet/Entity를 사용한다.

| OData | 용도 |
|---|---|
| `RFQHeaderSet` | RFQ Header 목록/KPI 기준 데이터 |
| `RFQItemSet` | 선택 RFQ의 Item 목록 |
| `MQCompareSet` | 선택 RFQ Item의 MQ 비교 목록 |
| `MQDetailSet` | MQ 상세 팝업 |
| `QuotationItemSet` | MQ 채택/채택취소 MERGE 처리 |

채택/채택취소 호출 구조:

```text
MERGE /QuotationItemSet(MqNo='...',MqItem='...')
Body:
{
  "MqNo": "...",
  "MqItem": "...",
  "ActionType": "AWARD" | "CANCEL"
}
```

## 7. 다음 개발자가 바로 확인할 테스트 케이스

1. 조회 버튼 클릭 후 Begin 컬럼에서 `RFQ KPI 요약`과 `RFQ 헤더 목록`이 위아래 두 블록으로 분리되는지 확인한다.
2. KPI 숫자를 기억한 뒤 `미채택 건수` 카드를 누른다.
3. RFQ Header Table만 미채택으로 필터링되고 KPI 숫자는 그대로인지 확인한다.
4. 상단 조회조건의 `채택상태` 값이 KPI 클릭 때문에 바뀌지 않는지 확인한다.
5. KPI 필터 해제 아이콘 클릭 시 Table이 마지막 조회 결과 전체로 돌아오는지 확인한다.
6. RFQ Header Table 우측의 정렬/그룹 아이콘을 눌러 Dialog가 열리는지 확인한다.
7. 채택상태 그룹 또는 문서일자 정렬을 적용해 표시가 바뀌는지 확인한다.
8. 정렬/그룹 초기화 버튼으로 기본 표시 상태로 돌아오는지 확인한다.
9. RFQ Header 행 앞쪽 highlight가 상태별로 표시되는지 확인한다.

## 9. 주의사항

- KPI 빠른 필터는 조회조건이 아니다. 조회조건 모델을 바꾸면 KPI 숫자가 다시 줄어드는 문제가 재발한다.
- RFQ Header Table의 `(N)`은 현재 화면에 보이는 행 수로 갱신한다.
- KPI 숫자는 마지막 조회 결과 기준이다.
- Search Help는 아직 구현하지 않는다. CDS View를 만든 후 `TableSelectDialog`로 연결한다.
- CSS 추가는 최대한 피하고 SAPUI5 표준 컨트롤과 layout 속성으로 해결한다.
- 이후 커밋 메시지는 한글로 작성하고 끝에 `_YYYYMMDD` 형식 날짜를 붙인다.
